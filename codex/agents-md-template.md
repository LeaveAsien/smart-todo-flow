## Project Management

This project uses three documents to manage the task workflow. Paste these rules into the target project's Codex `AGENTS.md` or `AGENT.md`:

| Document | Role | When to update |
|----------|------|----------------|
| `PLAN.md` | High-level plan: goals, phases, direction | When direction changes |
| `TODO.md` | Execution checklist: concrete next steps | Generated by `/todo`; mark `[x]` on completion |
| `CHANGELOG.md` | Change log: record of every change | Append immediately after each TODO item is done |

### Workflow Rules

- When discussion produces **directional/planning content** (multiple directions, phase breakdowns, goal visions) -> write to PLAN.md first, then pick items to generate TODO
- After completing a TODO item -> immediately append to CHANGELOG.md; group related changes under the same heading
- CHANGELOG format: `## YYYY-MM-DD (batch summary)` + `- <verb> <description>`
- For detailed TODO state management (generation, resuming, phase flow), see `/todo`
